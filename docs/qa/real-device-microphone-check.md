# 실기기 마이크 점검 체크리스트 (교사용)

Date: 2026-05-09
App: https://wbmaker2.github.io/poetry-bgm-studio/
목적: 수업 전 교사용 실제 기기에서 마이크, 녹음, 믹싱, 저장이 정상 동작하는지 점검하고, 공유 기기에서의 개인정보 주의사항을 확인한다.

## 1) 점검 전 준비

1. 앱을 **HTTPS** 주소로 접속한다.
   - 교실 기기에서는 `http://`보다 `https://wbmaker2.github.io/poetry-bgm-studio/` 사용이 안정적이다.
2. 브라우저가 **마이크 접근 권한 요청**을 보여줄 수 있도록 사이트 설정을 초기화해 둔다.
3. 이어폰 또는 외부 스피커를 사용 중이면, 녹음 중 모니터링이 들려서 학생이 피드백을 받기 쉽게 배치한다.
4. 수업 전 배경음악 파일(`public/audio/*.wav`)이 로드되는지 확인한다.

## 2) 브라우저별 실기기 점검 (Chrome / Edge / Safari / Chromebook)

### 공통 항목

- 녹음 패널의 상태 문구가 실제 동작과 일치하는지 확인한다.
  - `마이크 권한을 요청하고 있습니다.`
  - `녹음 중입니다. 시를 천천히 낭송해 보세요.`
  - `녹음이 저장되었습니다. 배경음악을 골라 들어볼 수 있습니다.`
  - `마이크를 사용할 수 없습니다. 브라우저 권한을 확인해 주세요.`
- `낭송 녹음 시작` 버튼을 누른 뒤 권한 허용 요청이 뜨면 **허용**을 선택한다.
- 권한이 거부된 경우, 브라우저 설정에서 해당 사이트 권한을 허용으로 변경하고 페이지를 새로고침한다.

### Chrome

1. 주소창 왼쪽 자물쇠/권한 표시 → 마이크를 `허용` 상태로 고정한다.
2. 앱에서 `낭송 녹음 시작`을 누르고 3~5초 정도 읽는다.
3. `녹음 정지` 후 `내 목소리 들어보기`를 눌러 녹음본 재생이 되는지 확인한다.
4. 사운드 팔레트 버튼 중 하나(`빗소리`, `잔잔한 피아노`, `파도 소리`, `새소리`, `시냇물 소리`, `바람 소리`, `오르골`, `종소리`)를 선택해 음색이 함께 믹스되는지 확인한다.
5. 사운드는 최대 2개까지만 선택되는지 확인한다.
6. `오디오 엽서 저장` 클릭 후 `.wav` 파일이 다운로드되는지, 파일명이 `poetry-bgm-studio-YYYYMMDD-HHMMSS.wav` 형식인지 점검한다.

### Edge

1. 권한 안내가 Chrome과 동일하게 자물쇠/사이트 설정에서 마이크 허용을 확인한다.
2. 학교 계정 로그인 상태의 정책 제약이 있는 기기라면 정책으로 마이크가 차단되어 있지 않은지 점검한다.
3. 위 공통 항목(`낭송 녹음 시작`/`녹음 정지`/`내 목소리 들어보기`/선택/저장)을 동일하게 수행한다.

### Safari

1. macOS/IOS Safari에서 처음 접속 시 상단 또는 주소창 옆 권한 다이얼로그가 뜨는지 확인한다.
2. `시스템 환경설정 > 개인정보 보호 및 보안 > 마이크`(macOS)에서 브라우저/사이트 권한을 허용한다.
3. `낭송 녹음 시작`, `녹음 정지`, `내 목소리 들어보기`, 사운드 팔레트 선택, WAV 저장을 동일하게 수행한다.

### Chromebook

1. Chrome 기반 Chromium에서 `settings > privacy and security > site settings > microphone`에 접근해 사이트 권한이 허용인지 확인한다.
2. 원격 관리/관리자 정책이 마이크를 제한하지 않는지 학급 단말 정책에서 확인한다.
3. 녹음 장치(내장 마이크/외장 마이크) 선택이 올바른지 점검한 뒤 공통 항목을 수행한다.

## 3) 점검 체크리스트 (실제 수업 시작 전)

- [ ] HTTPS 주소에서 앱이 정상 로드됨
- [ ] 마이크 권한을 허용했을 때 `낭송 녹음 시작` 버튼이 활성화됨
- [ ] `낭송 녹음 시작` 버튼 동작 확인
- [ ] `녹음 정지` 버튼 동작 확인
- [ ] `내 목소리 들어보기` 동작 확인
- [ ] `다시 녹음` 버튼으로 현재 녹음이 초기화되는지 확인
- [ ] 사운드 팔레트 8종 선택 토글 동작 확인
- [ ] 사운드가 최대 2개까지만 선택되는지 확인
- [ ] `오디오 엽서 저장` 클릭 시 WAV 생성/다운로드 확인
- [ ] 저장된 파일이 `.wav` 포맷인지 확인
- [ ] 다운로드 위치(브라우저 기본 다운로드 폴더 또는 지정 폴더)가 수업용 PC/크롬북에서 확인됨
- [ ] 재현 가능한 문제(권한 거부, 무음, 저장 실패)가 있으면 즉시 사유 기록

## 4) 개인정보·공유 기기 주의 안내

- 실기기 검증은 교사가 직접 수행하고, 학생 계정이 아닌 교사용 단말에서 먼저 확인한다.
- 마이크 테스트는 수업이 시작되기 전, 혹은 휴식 시간에 수행해 학생의 발화 녹음이 불필요하게 늘어나지 않게 한다.
- Chrome/Edge/Safari/Chromebook 모두, 마이크 권한은 수업 시작 전 꼭 허용 상태인지 다시 확인한다.
- 공용 태블릿/컴퓨터에서는 **헤드셋 사용을 권장**하고, 주변학생과 목소리가 함께 녹음되지 않도록 좌우 배치 및 거리 조절을 한다.
- 다운로드 파일은 클래스별 폴더(또는 지정된 교사용 폴더)에만 저장하고, 불필요한 노출이 생기지 않도록 즉시 정리한다.
- 앱 자체는 녹음 파일을 서버로 업로드하지 않지만, 다운로드 폴더가 iCloud/Drive/OneDrive/학교 MDM 등과 동기화될 수 있으므로 수업 전 저장 위치와 학교 개인정보 지침을 확인한다.
- 수업 종료 후에는 학교의 파일 보관/공유 정책에 맞춰 필요한 파일만 남기고 선별 삭제한다.
